# Pass/fail

**Purpose:** Cohort pass/fail determination. No letter grades. Fail → free re-enrollment ([guarantee-refunds-buyouts.md](../business/guarantee-refunds-buyouts.md)).

---

## Pass bar

Pass requires **all** of the following:

### Phase 1 (6 checks)

| Unit | Gate |
|------|------|
| Project 1 reviews | 29/29 submitted |
| Project 2 reviews | 29/29 submitted |
| Project 3 reviews | 29/29 submitted |
| Votes | 3/3 ballots submitted |
| Builds | ≥ **2/3** builds eligible at deadline |
| Developer/user cycles | ≥ **2/3** four-week cycles meet minimums ([metrics.md](metrics.md)) |

### Phase 2 (3 checks)

| Unit | Gate |
|------|------|
| Learning app | ≥ 25 external users (platform snapshot) |
| Venture | ≥ 25 users + ≥ 1 investor touchpoint |
| Open source | ≥ 1 merged PR in qualified repo |

### Participation

| Gate | Threshold |
|------|-----------|
| Mandatory checkpoints | ≤ 20% missed ([guarantee-refunds-buyouts.md](../business/guarantee-refunds-buyouts.md)) |
| Week 1 setup | Completed |

**No holistic staff override for fail → pass.** Staff may note distinction.

---

## Pass vs job-offer ready

| | Pass | Job-offer ready |
|---|------|-----------------|
| **Gate** | Metric thresholds above | Pass + [job-offer-readiness.md](job-offer-readiness.md) checklist |
| **When** | Week 16 | Week 16 + rolling |
| **Effect** | Certificate + placement pool | Active partner promotion |

Student may **pass but not yet ready** — still in placement pool, guarantee active.

---

## Partial completion

| Scenario | Result |
|----------|--------|
| Crushed Phase 1, failed Phase 2 metric | **Fail** — re-enroll free |
| Failed 1 review set | **Fail** that unit → fail cohort |
| 2/3 builds only | **Pass** (if all else met) |
| Strong Phase 2, weak Phase 1 reviews | **Fail** |

No partial pass certificate cohort 1.

---

## Failure communication

| When | Action |
|------|--------|
| Dec 19, 18:00 | Private email: pass or fail + metric breakdown |
| Dec 20 | Re-enrollment form link (Spring 2027 cohort) |
| Dec 20 | Pass students: placement kickoff calendar invite |

Appeals: 48 hr window; program director reviews only for metric collection errors (GitHub API bug, etc.).
