# Comms platform: review rubric

Uses standard format from [peer-review-system.md](../../../assessment/peer-review-system.md). **Dimension 5 swapped** for comms-specific criterion.

---

## Rubric dimensions (1–5)

| # | Dimension | 5 = excellent |
|---|-----------|---------------|
| 1 | Production readiness | 15 users chatting simultaneously without loss |
| 2 | Core functionality | All baseline comms features work |
| 3 | Code quality | Clear realtime architecture; tested message paths |
| 4 | Ecosystem thinking | Designed for 30-person cohort noise; moderation; onboarding |
| 5 | **Reliability & realtime UX** *(replaces generic UX)* | Messages instant; notifications reliable; search works |

### Dimension 5 detail

| Score | Criteria |
|-------|----------|
| 1 | Messages lost; refresh required |
| 3 | Works but laggy or notifications inconsistent |
| 5 | Feels like Slack/Discord replacement for daily use |

---

## Review logistics (2-day window)

- Mon 14:00: reviews due (29/29)
- Tue 16:00: vote closes
- Shorter depth acceptable on non-assigned reviews (≥ 120 words min this project only)

Votes and tie-break: same as Project 1.
