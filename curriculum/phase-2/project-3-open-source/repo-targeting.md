# Open source: repo targeting

---

## Qualifying repo criteria (checked at PR open)

| Criterion | Threshold |
|-----------|-----------|
| GitHub stars | ≥ **1,000** |
| OR active contributors | ≥ **50** in last 90 days |
| OR allowlisted | In `hult-cohort-{term}/qualified-repos.md` |

Allowlist for repos with strong merge culture but lower stars (e.g. 200 stars, fast merges) — program director maintains, ≥ 10 repos.

---

## Targeting strategy

1. **Research agent:** Map repo structure, CONTRIBUTING.md, CI, recent merge velocity
2. Pick issue labeled `good first issue` OR self-file bug with reproduction
3. Comment on issue before coding ("I'd like to work on this")
4. Prefer repos where median merge time < 14 days (check merged PR timestamps)

---

## Portfolio approach

| Parallel PRs | Recommendation |
|--------------|----------------|
| 1 active PR | Minimum |
| 2–3 active | Ideal — if one stalls, others progress |
| > 3 | Diminishing returns during semester |
