# Governance

How winners are selected, teams are formed, leadership is held conditionally, and recognition credentials are issued.

## Components

- [winner-selection.md](winner-selection.md) — voting mechanics for the three Phase 1 contests
- [team-formation.md](team-formation.md) — winners draft 10% of the cohort each
- [removal-succession.md](removal-succession.md) — leadership is conditional; paths up and out
- [credentials.md](credentials.md) — the Law Review-style recognition and how it's verified

## Principles (decided)

- Leadership is earned by winning open peer-judged contests, never appointed
- Leadership is conditional: teams can underperform, leaders can be removed or step down, others can step up
- There is a standing incentive for winners to stay on the straight and narrow
- All governance leaves a GitHub trail like everything else in the program
